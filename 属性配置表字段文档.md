# 属性配置表字段文档

## 1. 文档目标

这份文档用于把属性系统从“规则描述”推进到“程序建表与配表”层级。

目标是固定以下内容：

- 角色基础属性表结构
- 等级成长表结构
- 装备主模板与装备词条表结构
- 被动魂石表结构
- 技能公式字段表结构
- 属性汇总与结算所需的数据对象结构

本文件默认服务端为权威端，客户端只做展示和可选预览。

---

## 2. 总体设计原则

第一版建议遵循四条原则：

- 配置表字段使用稳定英文名，文档解释使用中文
- 配置表尽量拆成“主表 + 子表”，避免一张超宽表
- 枚举值固定，不使用自由文本
- 固定值与百分比字段分开，不混用

额外硬规则：

- 装备永远不提供生命属性
- 生命只允许来自角色基础、等级成长、天赋和系统固定成长
- 所有百分比字段统一用小数表示

例如：

- `15%` 记录为 `0.15`
- `120%` 记录为 `1.20`

## 3. 属性枚举定义

建议先固定一组全局属性枚举，所有表通过 `attribute_code` 关联。

### 3.1 第一版属性代码

- `health`
- `stamina`
- `mana`
- `move_speed`
- `physical_attack`
- `magic_attack`
- `physical_defense`
- `magic_defense`
- `physical_crit`
- `magic_crit`
- `crit_damage_bonus`
- `damage_bonus`
- `extra_damage`
- `crit_resist`
- `damage_immunity`
- `extra_immunity`
- `heal_power`
- `heal_taken_bonus`

### 3.2 推荐属性元数据表

表名建议：`attribute_defs`

字段：

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `attribute_code` | string | 是 | 属性唯一代码，主键 |
| `display_name` | string | 是 | 中文显示名 |
| `category` | enum | 是 | `base` / `attack` / `defense` / `healing` |
| `value_kind` | enum | 是 | `flat` / `ratio` |
| `default_value` | decimal | 是 | 默认值 |
| `min_value` | decimal | 否 | 最小允许值 |
| `max_value` | decimal | 否 | 最大允许值 |
| `client_visible` | bool | 是 | 是否展示在客户端面板 |
| `description` | string | 否 | 描述 |

## 4. 角色基础属性表

角色基础属性只定义“裸角色初始值”。

表名建议：`character_base_stats`

### 4.1 主键建议

- 若全角色共用一套基础值，主键可使用 `profile_id`
- 若按种族、模板、开局路线区分，可增加 `template_code`

第一版建议先用单套：

- `profile_id = default_player`

### 4.2 字段结构

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `profile_id` | string | 是 | 主键 |
| `health` | int | 是 | 初始生命 |
| `stamina` | int | 是 | 初始耐力上限 |
| `mana` | int | 是 | 初始法力上限 |
| `move_speed` | decimal | 是 | 初始移速 |
| `physical_attack` | int | 是 | 初始物理攻击 |
| `magic_attack` | int | 是 | 初始法术攻击 |
| `physical_defense` | int | 是 | 初始物理防御 |
| `magic_defense` | int | 是 | 初始法术防御 |
| `physical_crit` | decimal | 是 | 初始物理暴击 |
| `magic_crit` | decimal | 是 | 初始法术暴击 |
| `crit_damage_bonus` | decimal | 是 | 初始爆伤加成 |
| `damage_bonus` | decimal | 是 | 初始伤害加成 |
| `extra_damage` | decimal | 是 | 初始追加伤害加成 |
| `crit_resist` | decimal | 是 | 初始暴击抵抗 |
| `damage_immunity` | decimal | 是 | 初始伤害免疫 |
| `extra_immunity` | decimal | 是 | 初始追加免疫 |
| `heal_power` | decimal | 是 | 初始治疗强度 |
| `heal_taken_bonus` | decimal | 是 | 初始受治疗加成 |

### 4.3 第一版建议

- 初始暴击、爆伤、增伤、免疫类属性建议默认 `0`
- `physical_attack` 和 `magic_attack` 可保留基础值，避免 1 级裸装伤害过低

## 5. 等级成长表

等级成长用于定义角色每级自动增长的属性。

表名建议：`level_growth_stats`

### 5.1 字段结构

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `level` | int | 是 | 主键，等级 |
| `health_gain` | int | 是 | 本级生命增量 |
| `stamina_gain` | int | 是 | 本级耐力增量 |
| `mana_gain` | int | 是 | 本级法力增量 |
| `physical_attack_gain` | int | 是 | 本级物攻增量 |
| `magic_attack_gain` | int | 是 | 本级法攻增量 |
| `physical_defense_gain` | int | 是 | 本级物防增量 |
| `magic_defense_gain` | int | 是 | 本级法防增量 |
| `move_speed_gain` | decimal | 是 | 本级移速增量 |

### 5.2 计算口径

角色某等级最终成长值建议为：

```text
当前等级成长属性 = 1 级到当前等级所有 gain 累加
```

### 5.3 可选扩展

如果后续要按阶段成长曲线切分，可新增：

- `growth_group`
- `world_tier`

第一版先不加。

## 13. 技能公式字段表

这一部分是程序实现伤害、治疗、追加伤害的核心。

### 13.1 公式主表

表名建议：`skill_formula_defs`

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `formula_id` | string | 是 | 主键 |
| `name` | string | 是 | 公式名 |
| `formula_type` | enum | 是 | 见 13.2 |
| `attack_attribute` | enum | 否 | 读取攻击属性 |
| `flat_value` | decimal | 是 | 固定值 |
| `ratio_value` | decimal | 是 | 倍率 |
| `extra_ratio_value` | decimal | 否 | 追加倍率 |
| `heal_ratio_value` | decimal | 否 | 治疗倍率 |
| `damage_bonus_enabled` | bool | 是 | 是否吃伤害加成 |
| `crit_enabled` | bool | 是 | 是否允许暴击 |
| `defense_enabled` | bool | 是 | 是否读取防御 |
| `immunity_enabled` | bool | 是 | 是否读取伤害免疫 |
| `random_enabled` | bool | 是 | 是否随机浮动 |
| `extra_chain_enabled` | bool | 是 | 是否允许触发追加链 |
| `description` | string | 否 | 说明 |

### 13.2 `formula_type` 枚举

- `main_damage`
- `extra_damage`
- `direct_heal`
- `heal_over_time`
- `life_steal`
- `resource_restore`

### 13.3 `attack_attribute` 枚举

- `physical_attack`
- `magic_attack`
- `none`

### 13.4 第一版公式固定口径

对照当前玩法文档，第一版建议：

- `main_damage`：吃伤害加成、可暴击、吃防御、吃伤害免疫、可随机
- `extra_damage`：不吃伤害加成、不暴击、吃防御、吃追加免疫、不随机
- `direct_heal`：不暴击、不吃防御、不吃免疫、可随机
- `heal_over_time`：不暴击、不吃防御、不吃免疫、默认不随机
- `life_steal`：基于最终伤害后处理，不单独读防御

| `effect_type` | enum | 是 | `dot` / `hot` / `shield` / `trigger` |
| `formula_id` | string | 否 | 关联公式 |
| `payload_json` | json | 否 | 参数 |

## 16. 现有协议结构的演进建议

当前仓库中的 `CharacterStats` 结构较简化，只分：

- `base`
- `attack`
- `defense`

正式开发属性系统后，建议不要直接把所有字段平铺进旧结构，而是升级为：

### 16.1 存档层结构

建议存档只保存：

- 角色等级
- 已激活天赋
- 当前装备
- 当前被动魂石装配
- 当前 Buff 不持久化，通常不入档

最终属性不要直接持久化为权威真值。

### 16.2 传输层结构

协议返回给客户端时，建议使用：

- `base_stats`
- `derived_stats`
- `combat_stats`

其中：

- `base_stats` 给基础类
- `derived_stats` 给成长类和装备类结果
- `combat_stats` 给战斗实际使用属性

### 16.3 推荐 JSON 结构

```json
{
  "baseStats": {
    "health": 1200,
    "stamina": 100,
    "mana": 80,
    "moveSpeed": 4.2
  },
  "combatStats": {
    "physicalAttack": 260,
    "magicAttack": 120,
    "physicalDefense": 180,
    "magicDefense": 90,
    "physicalCrit": 0.18,
    "magicCrit": 0.05,
    "critDamageBonus": 0.30,
    "damageBonus": 0.12,
    "extraDamage": 0.10,
    "critResist": 0.06,
    "damageImmunity": 0.08,
    "extraImmunity": 0.00,
    "healPower": 0.15,
    "healTakenBonus": 0.10
  }
}
```

## 17. 启动校验规则

程序启动加载配置时，建议统一执行配置校验。

至少检查：

- 装备属性表中不能出现 `health`
- 被动魂石属性表中不能出现 `health`
- 公式表中 `main_damage` 必须指定攻击属性
- 公式表中 `direct_heal` 必须指定治疗属性
- 主动魂石必须至少有一条效果线
- `supported_weapon_types` 不能为空
- 所有外键都能在主表找到

## 19. 第一版结论

这份字段文档固定了属性系统的核心建表方向：

- 属性统一走 `attribute_code`
- 角色成长拆成基础、等级、天赋、装备、被动魂石、Buff 六层
- 技能和治疗统一通过公式表驱动
- 装备和被动魂石都严禁提供生命属性
- 程序应保存来源分层，而不是只保存一个最终面板

按这份文档落地后，后续可以直接继续写：

- SQL 表结构初稿
- Go 配置结构体
- 配置导表格式
- 客户端属性面板协议
