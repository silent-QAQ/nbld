# 团结引擎客户端协同开发约束

本项目客户端统一使用 `团结引擎 + C#`。

## 统一规则

- 所有客户端开发者统一使用团结引擎
- 不再使用标准 Unity 打开同一客户端工程
- 团队内统一编辑器版本
- 团队内统一插件版本与导入设置
- 服务端接口变更后，先更新 `shared/proto`

## 当前目录约定

- 客户端目录仍保留为 `client/unity`
- 该目录名仅为历史命名，不表示允许回切标准 Unity

## 首次接入顺序

1. 在本地安装团结引擎
2. 在 `client/unity` 下创建客户端工程
3. 导入当前仓库已有脚本：
   - `Assets/Scripts/Protocol`
   - `Assets/Scripts/Network`
   - `Assets/Scripts/World`
4. 创建最小登录/世界场景
5. 将 `WorldBootstrap` 挂到场景对象
6. 把 `Http Base Url` 指向可访问的 Go 服务入口
7. 进入 Play 模式验证登录、进世界、移动同步

## 协同方式

- 云服务器负责运行 Go 服务端、自检脚本、接口测试
- 本地团结引擎负责场景、输入、表现、UI、客户端调试
- 若外部入口经 NAT 或反向代理暴露，参考 `docs/deployment/cloud-nat.md`
