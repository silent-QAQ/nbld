# H5 客户端协同开发约束

本项目客户端统一使用 `H5 + TypeScript`。

## 统一规则

- 所有客户端开发者统一使用 H5 路线
- 优先使用代码驱动、可审查、可自动化的前端方案
- 服务端接口变更后，先更新 `shared/proto`
- 客户端默认面向 `PC 浏览器 + 手机浏览器`，后续再考虑 `PWA` 与桌面封装

## 当前目录约定

- 当前主客户端目录为 `client/web`
- `client/unity` 仅作历史原型保留，不再继续开发

## 首次接入顺序

1. 在 `client/web` 初始化 H5 客户端工程
2. 建立最小登录页与世界页
3. 接入当前仓库已有协议：
   - `shared/proto`
   - `HTTP` 登录接口
   - `WebSocket` 世界连接
4. 完成玩家输入、相机、区块占位渲染
5. 把 `Http Base Url` 指向可访问的 Go 服务入口
6. 在浏览器中验证登录、进世界、移动同步

## 协同方式

- 云服务器负责运行 Go 服务端、自检脚本、接口测试
- 本地浏览器负责输入、表现、UI、客户端调试
- 若外部入口经 NAT 或反向代理暴露，参考 `docs/deployment/cloud-nat.md`
