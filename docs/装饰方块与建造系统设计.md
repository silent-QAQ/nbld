# 装饰方块与建造系统设计

## 核心原则

**地形层（Terrain Layer）保持不变，不开发破坏机制。** 我们只关注：
1. **装饰层方块**（Decoration Layer）的采集工具系统
2. **建造系统**（Building System）玩家放置的建筑方块

---

## 系统架构

### ChunkTile 结构（当前协议）

```go
type ChunkTile struct {
    X          int    `json:"x"`
    Y          int    `json:"y"`
    Terrain    string `json:"terrain"`        // ❌ 不开发（生物群系）
    Block      string `json:"block"`          // ❌ 不开发（地形主块）
    Feature    string `json:"feature"`        // ❌ 不开发（河流等）
    Decoration string `json:"decoration"`    // ✅ 采集系统
    LevelHint  string `json:"levelHint"`      // （参考）
}
```

### 建造方块存储（新增）

```go
type PlacedBlock struct {
    ID        uuid.UUID
    MapID     string
    X, Y      int
    BlockID   string    // 如 "fence_wood", "wall_stone"
    OwnerID   uuid.UUID
    CreatedAt time.Time
}
```

## 第四部分：系统集成方案

### 数据库设计

#### 表：placed_blocks（玩家放置的建造方块）

```sql
CREATE TABLE placed_blocks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    map_id VARCHAR(32) NOT NULL,      -- 所在地图
    block_id VARCHAR(64) NOT NULL,    -- 建造方块ID
    x INT NOT NULL,                   -- 世界坐标 X
    y INT NOT NULL,                   -- 世界坐标 Y
    owner_id UUID NOT NULL REFERENCES characters(id),
    created_at TIMESTAMP NOT NULL DEFAULT now(),
    updated_at TIMESTAMP NOT NULL DEFAULT now(),
    destroyed_at TIMESTAMP,           -- NULL = 未破坏，有值 = 已破坏时间
    
    UNIQUE(map_id, x, y)              -- 每个位置最多一个方块
);

CREATE INDEX idx_placed_blocks_map_owner ON placed_blocks(map_id, owner_id);
CREATE INDEX idx_placed_blocks_coord ON placed_blocks(map_id, x, y);
```

### Go 代码结构

#### 新增文件：`server/internal/app/decoration_system.go`

```go
package app

type DecorationDefinition struct {
    Block      string  // 装饰方块ID如 "tree_deciduous"
```

## 总结

| 项 | 数量 | 说明 |
|----|------|------|
| **装饰层方块** | ~68 | 地图生成，可采集 |
| **采集工具** | 9 | 从赤手到铁工具 |
| **建造方块** | 42 | 玩家合成并放置 |
| **建造配方** | 25+ | 工作台合成 |
| **生物群系** | 25 | 装饰分布范围 |

**核心设计原则：**
- ✅ 装饰层采集与工具进度
- ✅ 建造方块合成与放置
- ✅ 物理属性（碰撞、透光、耐久）
- ❌ 地形层不开发破坏
