# 开发规范

## 总原则

- 先交付可运行结果，再追求理想设计
- 严格按阶段推进，不跨阶段堆功能
- 优先模块化，不做巨石文件
- 服务端权威
- Rust 仅用于已确认热点

## 目录约束

- 客户端代码放在 `client/unity`
- 服务端代码放在 `server`
- 共享协议定义放在 `shared/proto`
- 项目文档统一放在 `docs`
- 自动化脚本统一放在 `scripts`

## 服务端约束

- 服务端先以模块化单体实现，再按需要拆分服务
- `cmd` 放入口程序
- `internal` 放业务实现
- 任何网络协议变更先更新 `shared/proto`

## 客户端约束

- 客户端统一使用 `团结引擎 + C#`
- `client/unity` 目录名仅作历史兼容保留
- 禁止标准 Unity 与团结引擎混用同一客户端工程
- 客户端开发者需统一团结引擎版本与插件版本
- 客户端只负责表现、输入、局部预测和 UI
- 战斗、掉落、建造合法性、捕宠结果均以服务端为准

## 提交流程

- 每完成一个阶段目标，更新 `docs/tasks/status.md`
- 启动方式必须沉淀到 `scripts`
- 新模块必须说明输入、输出和后续验收方式
